# ANÁLISE E DEDUÇÃO DE FÓRMULAS DO DRE

## 1. ESTRUTURA GERAL DO DRE

O DRE está organizado em camadas hierárquicas que seguem a estrutura contábil clássica, partindo da **Receita Bruta** até o **Lucro Líquido** e **Fluxo de Caixa**. Cada linha representa um componente calculável que se relaciona com as métricas do Business Plan.

---

## 2. ANÁLISE POR BLOCO

### 2.1 BLOCO DE RECEITA (Revenue)

#### **Revenue Total**
**Fonte de Dados:** `monthData.totalRevenue` (calculations.ts)

**Fórmula Deduzida:**
```
Revenue[mês] = totalNewRevenue + totalRenewalRevenue + totalExpansionRevenue + totalLegacyRevenue
```

**Componentes:**
- **totalNewRevenue:** Ativações de novos clientes no mês (Activation)
- **totalRenewalRevenue:** Renovações de contratos Executar (Loyalty e No-Loyalty)
- **totalExpansionRevenue:** Expansões de clientes ativos + base legada
- **totalLegacyRevenue:** Receita da base legada após churn e expansão

**Observações do DRE Atual:**
- Janeiro: R$ 3.152.393,18 = Activation + Activation-Legado + Renewall + Renewall-Legado + Expansion + Expansion-Legado
- A receita é **cohortizada** por origem (Activation nova vs Legado)

---

### 2.2 BLOCO DE ACTIVATION (Receita de Novos Clientes)

#### **Activation**
**Fonte de Dados:** `monthData.totalNewRevenue` (apenas novos clientes ativados no mês)

**Fórmula Deduzida:**
```
Activation[mês] = Σ (Clientes_Ativados[tier][produto] × Ticket[tier][produto] × Duração_Contrato[produto])
```

**Detalhamento por Produto:**
- **Saber:** `clientes × ticket × revenueActivationRate` (projeto único)
- **Ter:** `clientes × ticket × revenueActivationRate` (projeto único)
- **Executar No-Loyalty:** `clientes × ticket × 2 meses` (contrato 2 meses)
- **Executar Loyalty:** `clientes × ticket × 7 meses` (contrato 7 meses)

**Subdivisões do DRE (para rastreabilidade):**
```
# [Saber] Revenue Won = Σ revenueByTierProduct[tier].saber
# [Ter] Revenue Won = Σ revenueByTierProduct[tier].ter
# [Executar-no loyalty] Revenue Won = Σ revenueByTierProduct[tier].executarNoLoyalty
# [Executar-loyalty] Revenue Won = Σ revenueByTierProduct[tier].executarLoyalty
```

**Exemplo (Janeiro no DRE):**
```
Activation = R$ 750.000
  [Saber] Revenue Won = R$ 600.000
  [Ter] Revenue Won = R$ 79.000
  [Executar-no loyalty] Revenue Won = R$ 21.000
  [Executar-loyalty] Revenue Won = R$ 7.000
```

---

#### **Activation - Legado**
**Fonte de Dados:** Base legada inicial (apenas Mês 1)

**Fórmula Deduzida:**
```
Activation_Legado[mês=1] = Σ legacyBase[tier].revenue
Activation_Legado[mês>1] = 0
```

**No DRE:**
- Janeiro: R$ 2.286.393,18 (base legada de 228 clientes)
- Fevereiro em diante: R$ 0 (já está contabilizada)

---

### 2.3 BLOCO DE RENEWALL (Renovações)

#### **Renewall**
**Fonte de Dados:** `monthData.totalRenewalRevenue` (renovações de contratos Executar)

**Fórmula Deduzida:**
```
Renewall[mês] = Renewall_NoLoyalty + Renewall_Loyalty
```

**Renewall No-Loyalty:**
```
Para cada coorte de Executar No-Loyalty:
  Se (mês_atual - mês_ativação) % 2 == 0 E renovações < 5:
    Clientes_Renovando = clientes_coorte × 80%
    Receita = Clientes_Renovando × Ticket[tier].executarNoLoyalty × 2 meses
```

**Renewall Loyalty:**
```
Para cada coorte de Executar Loyalty:
  Se (mês_atual - mês_ativação) % 7 == 0 E renovações < 2:
    Clientes_Renovando = clientes_coorte × 30%
    Receita = Clientes_Renovando × Ticket[tier].executarLoyalty × 7 meses
```

**Subdivisões do DRE:**
```
# [Executar-no loyalty] Revenue Won = Σ renewalRevenue[tier].executarNoLoyalty
# [Executar-loyalty] Revenue Won = Σ renewalRevenue[tier].executarLoyalty
```

**Observações:**
- As renovações começam a aparecer após os primeiros contratos expirarem (março no DRE)
- No-Loyalty: ciclo de 2 meses, 80% renovação, até 5 vezes
- Loyalty: ciclo de 7 meses, 30% renovação, até 2 vezes

---

#### **Renewall - Legado**
**Fonte de Dados:** Base legada com churn mensal

**Fórmula Deduzida:**
```
Renewall_Legado[mês] = Base_Legada_Anterior[mês-1] × (1 - churn_rate)

Onde:
  churn_rate = 7% (0.07)
  Base_Legada_Anterior[mês=1] = Activation_Legado (R$ 2.286.393,18)
```

**Progressão:**
```
Mês 1: Base inicial = R$ 2.286.393,18 → Renewall = 0 (ainda não renovou)
Mês 2: R$ 2.286.393 × (1 - 0.07) = R$ 2.131.413,78
Mês 3: R$ 2.131.413 × (1 - 0.07) = R$ 1.990.587,36
...e assim sucessivamente
```

---

### 2.4 BLOCO DE EXPANSION (Expansões de Carteira)

#### **Expansion**
**Fonte de Dados:** `monthData.totalExpansionRevenue` (expansões da base ativa)

**Fórmula Deduzida:**
```
Expansion[mês] = Expansion_Base_Ativa + Expansion_Legado

Expansion_Base_Ativa = Total_Clientes_Executar_Ativos × 5% × Distribuição_Produtos[tier]
```

**Distribuição de Produtos por Tier:**
```
Large/Enterprise:
  Saber: 5%, Ter: 25%, Executar No-Loyalty: 70%

Medium:
  Saber: 5%, Ter: 40%, Executar No-Loyalty: 55%

Small/Tiny:
  Saber: 20%, Ter: 80%
```

**Cálculo Detalhado:**
```
Para cada tier:
  Clientes_Ativos = Σ activeExecutarLoyalty[tier] + activeExecutarNoLoyalty[tier]
  Clientes_Expandindo = round(Clientes_Ativos × 5%)
  
  Para cada produto:
    Clientes_Produto = round(Clientes_Expandindo × Distribuição[tier][produto])
    Receita = Clientes_Produto × Ticket[tier][produto] × Duração[produto]
```

**Subdivisões do DRE:**
```
# [Saber] Revenue Won = Σ expansionRevenue[tier].saber
# [Ter] Revenue Won = Σ expansionRevenue[tier].ter
# [Executar-no loyalty] Revenue Won = Σ expansionRevenue[tier].executarNoLoyalty
```

---

#### **Expansion - Legado**
**Fonte de Dados:** Base legada com expansão de 5%

**Fórmula Deduzida:**
```
Expansion_Legado[mês] = Renewall_Legado[mês] × 5%

Distribuído por produtos conforme tabela de distribuição acima
```

**Exemplo (Janeiro):**
```
Base Legada Janeiro = R$ 2.286.393,18
Expansion = 0 (ainda não há renovação para expandir)

Fevereiro:
Renewall Legado = R$ 2.131.413,78
Expansion Legado = R$ 2.131.413,78 × 5% ≈ R$ 64.000 (aproximado)
```

---

### 2.5 CONVERSÕES (Saber → Executar)

**Fórmula Deduzida (não aparece como linha isolada no DRE, mas impacta Activation):**
```
Após 2 meses (60 dias) da ativação de Saber:
  Clientes_Convertendo = Clientes_Saber × 40%
  
  Loyalty = round(Clientes_Convertendo × 40%)
  No-Loyalty = Clientes_Convertendo - Loyalty (60%)
  
  Receita_Loyalty = Loyalty × Ticket[tier].executarLoyalty × 7 meses
  Receita_NoLoyalty = No-Loyalty × Ticket[tier].executarNoLoyalty × 2 meses
```

**Impacto:**
- Essas conversões entram como **Activation** no mês da conversão
- Exemplo: Cliente Saber ativado em Janeiro → converte em Março

### 2.6 MÉTRICAS CALCULADAS (Linhas de KPI)

#### **CAGR (Compound Annual Growth Rate)**
**Fórmula:**
```
CAGR = (Revenue[Dezembro] / Revenue[Janeiro])^(1/11) - 1
```

#### **CAC (Customer Acquisition Cost)**
**Fonte de Dados:** Investimento Marketing ÷ Clientes Novos

**Fórmula Deduzida:**
```
CAC[mês] = Investimento_Marketing[mês] / Novos_Clientes_Won[mês]

Onde:
  Investimento_Marketing = topline.investmentMonthly[mês]
  Novos_Clientes_Won = Σ wons[tier] (todas as tiers)
```

**Exemplo (Janeiro DRE):**
```
Investimento = R$ 400.000 (aproximado)
Clientes Won = 37 (soma de todos os tiers)
CAC = R$ 400.000 / 37 ≈ R$ 10.810,81
```

#### **CLV (Customer Lifetime Value)**
**Fórmula Deduzida:**
```
CLV[mês] = (Revenue[mês] / Quantidade_Clientes[mês]) × Vida_Média_Cliente / (1 + Taxa_Desconto)^meses

Onde:
  Vida_Média_Cliente depende do produto:
    - Executar Loyalty: 7 meses × (1 + taxa_renovação × renovações_max)
    - Executar No-Loyalty: 2 meses × (1 + taxa_renovação × renovações_max)
    - Saber/Ter: 1 projeto + eventual conversão
```

**No DRE Atual:**
- Janeiro: CLV = R$ 42.631,17
- O valor alto sugere inclusão de renovações e expansões
- Decresce ao longo do ano conforme base se normaliza

#### **ROI (Return on Investment)**
**Fórmula Deduzida:**
```
ROI[mês] = (CLV[mês] - CAC[mês]) / CAC[mês]

Ou acumulado:
ROI_Acumulado = (Receita_Total_Acumulada - Investimento_Total_Acumulado) / Investimento_Total_Acumulado
```

#### **Quantidade de Clientes**
**Fonte de Dados:** `monthData.totalActiveClients`

**Fórmula Deduzida:**
```
Quantidade_Clientes[mês] = Clientes_Ativos_Novos + Clientes_Legados_Ativos

Onde:
  Clientes_Ativos_Novos = Σ activeClients[tier][produto]
  Clientes_Legados_Ativos = Σ legacyClients[tier]
```

#### **Taxa de Desconto**
**Constante:** 1% ao mês (usado para calcular valor presente de receitas futuras)

#### **(%) Churn Mês**
**Constante:** 7% ao mês (aplicado à base legada)

### 2.10 TRIBUTOS E DEDUÇÕES


#### **(-) IMPOSTOS** (Regime de Tributação)
**Fórmula Deduzida:**
```
Impostos[mês] = ISS + IRRF + PIS + COFINS
```

**Detalhamento:**
- **ISS (Imposto Sobre Serviços):** 2% da Receita Bruta Recebida
- **IRRF (Imposto de Renda Retido na Fonte):** 1,5% da Receita Bruta Recebida
- **PIS (Programa de Integração Social):** 1,65% da Receita Bruta Recebida
- **COFINS (Contribuição para Financiamento da Seguridade Social):** 3,65% da Receita Bruta Recebida

**Soma Total de Impostos:** 2% + 1,5% + 1,65% + 3,65% = **8,8%**

**Fórmulas Individuais:**
```
ISS[mês] = Receita_Bruta_Recebida × 2%
IRRF[mês] = Receita_Bruta_Recebida × 1,5%
PIS[mês] = Receita_Bruta_Recebida × 1,65%
COFINS[mês] = Receita_Bruta_Recebida × 3,65%
```

**Exemplo (Janeiro):**
```
Receita Bruta Recebida: R$ 2.868.677,79

ISS = R$ 2.868.677,79 × 2% = R$ 57.373,56
IRRF = R$ 2.868.677,79 × 1,5% = R$ 43.030,17
PIS = R$ 2.868.677,79 × 1,65% = R$ 47.333,18
COFINS = R$ 2.868.677,79 × 3,65% = R$ 104.706,74

Total Impostos = R$ 252.443,65 (≈ 8,8%)
```

### 2.12 CUSTO DE SERVIÇO PRESTADO (CSP)

#### **% CSP (Margem de CSP)**
**Observação (Janeiro):** 35,71%

**Fórmula Deduzida:**
```
% CSP[mês] = CSP[mês] / Receita_Líquida[mês]
```

#### **(-) CUSTO SERVIÇO PRESTADO**
**Fonte de Dados:** Capacity Plan (headcount × custo médio por cargo)

**Fórmula Deduzida:**
```
CSP[mês] = CSP_Executar + CSP_Ter + CSP_Saber

Onde cada componente:
CSP_Produto[mês] = (CSP_Direto + CSP_Overhead + Outros)
```

**Componentes:**

##### **CSP EXECUTAR**
```
CSP_Executar_Direto = Σ (HC_Executar[cargo] × Custo_Cargo × Alocação_Direta%)
CSP_Executar_Overhead = Σ (HC_Executar[cargo] × Custo_Cargo × Alocação_Overhead%)
Outros_Executar = Custos variáveis (ferramentas, terceiros, etc.)
```

**Fonte de HC:** `capacityPlan.hcExecutar` (15 pessoas por squad)

##### **CSP TER**
```
CSP_Ter_Direto = Clientes_Ter_Ativos × Custo_Por_Cliente_Ter_Direto
CSP_Ter_Overhead = Clientes_Ter_Ativos × Custo_Por_Cliente_Ter_Overhead
Outros_Ter = Custos variáveis
```

##### **CSP SABER**
```
CSP_Saber_Direto = Clientes_Saber_Ativos × Custo_Por_Cliente_Saber_Direto
CSP_Saber_Overhead = Clientes_Saber_Ativos × Custo_Por_Cliente_Saber_Overhead
Outros_Saber = Custos variáveis
```

**Fonte de HC:** `capacityPlan.hcSaber` (9 pessoas por squad)

**Observação do DRE:**
- O CSP aumenta conforme o headcount cresce para atender mais clientes
- Há uma relação direta entre squads necessárias e CSP

## 8. FÓRMULAS CONSOLIDADAS (RESUMO)

```typescript
// RECEITA
revenue = totalNewRevenue + totalRenewalRevenue + totalExpansionRevenue + totalLegacyRevenue

// DEDUÇÕES
receitaBrutaRecebida = revenue × 0.91 // (1 - 4% - 3% - 2%)

// TRIBUTOS
royalties = receitaBrutaRecebida × 0.15
iss = receitaBrutaRecebida × 0.02
irrf = receitaBrutaRecebida × 0.015
pis = receitaBrutaRecebida × 0.0165
cofins = receitaBrutaRecebida × 0.0365
impostos = iss + irrf + pis + cofins

// RECEITA LÍQUIDA
receitaLiquida = receitaBrutaRecebida - royalties - impostos

// CSP
csp = (hcSaber × custoSquadSaber / 9) + (hcExecutar × custoSquadExecutar / 15)

// MARGENS
margemOperacional = receitaLiquida - csp
margemContribuicao = margemOperacional - despesasMarketingVendas
ebitda = margemContribuicao - despesasAdministrativas
ebit = ebitda - despesasFinanceiras + receitasFinanceiras
lucroLiquido = ebit - (ebit × 0.15) - (ebit × 0.09) // IRPJ e CSLL

// KPIS
cac = investimentoMarketing / Σ wons
clv = (receita / qtdClientes) × vidaMediaCliente / (1 + taxaDesconto)^meses
roi = (clv - cac) / cac
cagr = (revenueDez / revenueJan)^(1/11) - 1
```
